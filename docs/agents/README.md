# Agent Dev Environment

`docs/agents` фиксирует контекст, границы ответственности и правила работы для агентской разработки этого проекта.

## Цель

Дать любому агенту минимальный набор документов, чтобы он мог быстро войти в проект и делать изменения без нарушения архитектуры:

- `Python bot` — только Telegram UX и HTTP-клиент backend
- `Go backend` — вся бизнес-логика, БД, платежи, VPN, admin
- `deploy` — артефакты выката и окружение VPS

## Состав

- `architecture.md` — карта системы и слоёв
- `workflow.md` — как вести задачу агентом
- `contracts.md` — API и архитектурные инварианты
- `roles.md` — рекомендуемые зоны ответственности агентов
- `backlog.md` — ближайшие направления после MVP

## Минимальный ритуал перед задачей

1. Прочитать `architecture.md`.
2. Проверить `contracts.md`, чтобы не нарушить разделение bot/backend.
3. Обновить код в одной зоне ответственности за раз.
4. Прогнать доступную локально проверку.
5. Зафиксировать, что изменено и что осталось заглушкой.
