# Agent Workflow

## Как брать задачу

1. Определить зону: `backend-go`, `bot-python`, `deploy` или `docs`.
2. Проверить, не тянет ли задача изменения API-контракта.
3. Если контракт меняется, сначала обновить `contracts.md`.
4. Затем менять код.

## Как резать работу

- Backend agent:
  работает от DTO -> service -> handler -> route.
- Bot agent:
  работает от handler -> client method -> UX текстов/кнопок.
- Deploy agent:
  не меняет бизнес-логику, только запуск, конфиги, наблюдаемость.
- Docs agent:
  фиксирует текущую архитектуру, а не желаемую фантазию.

## Перед завершением

1. Проверить сборку или хотя бы синтаксис в своей зоне.
2. Указать, какие mock-части ещё не заменены реальными интеграциями.
3. Если добавлен новый env, обновить `.env.example`.
